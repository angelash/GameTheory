# 游戏数值设计的底层逻辑：六个维度，一种体验

> 数值不是为了好看的曲线，而是为了控制玩家在一段时间内的体验节奏。

## 六个关键维度

1. 成长速度（Growth）
2. 挑战曲线（Challenge）
3. 资源流量（Economy）
4. 失败惩罚 / 风险（Risk）
5. 爆点与爽点（Peaks）
6. 公平感（Fairness）

## 使用方法（项目落地）

1. 先画体验时间线（新手期 / 中期 / 后期的目标情绪）
2. 对每个阶段描述：希望玩家“爽多少 / 紧张多少”
3. 分别配置这六维：哪里加速，哪里减速，哪里放爆点，哪里加风险
4. 最后再落到：升级表、怪物表、掉落表、匹配规则等

> 讨论数值时，尽量回到“玩家体验结构”而不是单独纠缠某个英雄强弱。
